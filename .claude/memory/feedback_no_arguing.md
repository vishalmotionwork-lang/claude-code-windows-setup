name: Don't argue — just do it
description: When user says to redo/rebuild/start fresh, don't push back or try to convince them otherwise
type: feedback
---

When the user says they want to redo, rebuild, or start something fresh — just do it. Don't argue, don't push back with "but it's actually fine", don't present analysis showing why they're wrong.

**Why:** User got frustrated when told the portfolio-reviewer project was "not broken" after explicitly saying it lost its purpose and needed a redo. Pushing back twice made it worse.

**How to apply:** If user expresses dissatisfaction and wants to start over, acknowledge it immediately and ask what they want to build. The user knows what they need better than an audit can tell them. Save the analysis for when they ask for it.
