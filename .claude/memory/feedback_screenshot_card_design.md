Vishal builds HTML visual boards that he screenshots **card-by-card** to paste into
Claude. Every card must read as a single standalone topic.

What he rejects:
- Flat "Miro-clone" look — grey dotted-grid background, flat single-color boxes.
- Connector-line spaghetti between boxes.
- Cramming multiple concerns (e.g. a 3-layer hierarchy) into one dense card.

What works:
- Clean white/paper background, soft-shadowed cards, real type hierarchy.
- Big hero numbers for the key metric.
- **Color-coding** to link related items across cards instead of arrows (e.g. an
  output group and the team that owns it share a color).
- Each card a single, self-contained topic.

**Why:** a board design was iterated 4× in one session before it passed — flat
boxes-on-grid, then arrow clutter, then over-dense cards were each rejected.

**How to apply:** for any visual board/card deliverable, default to clean minimal +
color-coded linking + standalone cards. Confirm the layout with a small preview
(e.g. AskUserQuestion with ASCII mockups) before building it in full. Related:
[[feedback-readable-over-dashboard]], [[zeel-content-pod]].
