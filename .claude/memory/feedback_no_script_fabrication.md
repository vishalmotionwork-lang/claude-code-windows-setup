When writing video scripts (for Zeel or any creator), never invent specific stories, clients, numbers, timelines, or receipts to make the script "feel real." If the creator hasn't told me the real detail, the script MUST mark it `[ZEEL FILL: <what's needed>]` and move on — not fake it.

**Why:** User called out a v2 script draft on 2026-04-23 where I invented a "Paper Plane Coffee" client with fake Monday-to-Friday timeline ("8:15 AM, eleven minutes, 45 → 4 min, 40+ runs in 90 days"). User reaction: *"the script is very very mature like you have made assumption and that real case scenario things in each so made up."* Invented specificity is worse than generic — it sounds plausible but is false, and the creator has to either memorize fiction or rewrite the entire script on shoot day.

**How to apply:**
- Every number, client name, and timeline in a script draft must be either (a) sourced from a real input the user gave me, or (b) marked `[ZEEL FILL: specific thing creator needs to provide]` with a length budget (e.g., "40 sec demo")
- Never pad a script with fake texture to hit word-count targets — write tighter instead
- When a pattern from research (e.g., Jeff Su's format, Greg Isenberg's thesis-first shape) is being applied, teach the structure, not an invented story
- Structural writing (define → analogy → demo → rule) is preferred over narrative writing (client-story spine) when real receipts aren't available
- If the creator asks for their script to feel real, push back — ask for the real receipts first. Shoot day is where reality lands, not the draft
