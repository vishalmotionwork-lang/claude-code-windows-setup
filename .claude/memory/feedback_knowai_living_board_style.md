When building any KnowAI FigJam board (YouTube production, strategy, accountability), match the **existing living-board aesthetic**, not the generic FigJam sticky-note look.

The look (from the live KnowAI board):
- **Section headers** = rounded colored pill with a circled number + bold UPPERCASE title (e.g. "⑤ VIDEO TYPES — WHAT WE MAKE"). Blue / green / pink / purple families.
- **Section body** = large rounded rectangle, pale pastel fill, 2px border matching the header color.
- Inside: bold lead line + small gray sub-line, then **clean white sub-cards** (subtle border + soft shadow), short titles, small colored metadata/links.
- **Real screenshots embedded** (YT thumbnails, IG grid) with tiny monospace blue IDs underneath.
- **System sans font** (Inter), generous spacing, light dot-grid canvas.

**Why:** Vishal corrected this twice — rejected a dense HTML-table version AND a sticky-note + marker-font version. The team already runs a polished living board and new sections must look native to it.

**How to apply:** Before building, ask for or recall the reference board screenshot. Preview in HTML first in this exact style, get approval, THEN build in FigJam via [[reference_knowai_design_system]] + the figjam-board skill. Don't default to stickies for KnowAI. Reuse the placeholder-owner + DoD-first + max-2-WIP rules from the accountability board (KnowAI Development/LIVING-BOARD-HANDOFF.md).
